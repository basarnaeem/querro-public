## How screenshots are captured

Each screenshot follows three rules.

**Real data, sanitized.** Screenshots are taken against working test fixtures. Where data could identify a specific firm or transaction, that data is replaced with neutral placeholder values. The platform's interaction with real data is faithful; the values shown are not.

**Annotated for clarity.** Where a screenshot shows a non-obvious interaction or feature whose purpose is not visible at first glance, the image is annotated with brief callout text in a consistent style.

**Versioned by date.** Each filename includes the production date in `YYYY-MM-DD` form so readers can place it on the development timeline in [Roadmap](../docs/06-roadmap.md). Replaced screenshots are removed rather than retained, keeping this folder a current view rather than a historical archive.

## What this folder is not

This folder does not host marketing creative, brand assets, logo files, or design system documentation. Those live separately and are not part of the public documentation set. For brand asset inquiries, see [CONTRIBUTING.md](../CONTRIBUTING.md).
